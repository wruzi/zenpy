Syntax & Variables (Phase 0-1)

Use indentation, readable variable names, and explicit output formatting. This foundation maps to early ZenPy loops and condition drills.

username = "zen_coder"
level = 3
xp = 720

print(f"{username} | Lv{level} | XP {xp}")

Control Flow (Phase 1)

Use if/elif/else for branching and loops for deterministic computation. Prefer linear logic over deep nesting.

score = 78
if score >= 90:
    rank = "A"
elif score >= 70:
    rank = "B"
else:
    rank = "C"

for i in range(1, 4):
    print(i, rank)

Functions (Phase 2)

Write small, testable functions. Return values instead of printing from every helper.

def ap_nth(a: int, d: int, n: int) -> int:
    return a + (n - 1) * d

print(ap_nth(2, 3, 5))

Collections & Frequency Maps (Phase 3-4)

Lists support ordered iteration; dictionaries power frequency counting and fast lookups for many interview-style tasks.

text = "banana"
freq = {}
for ch in text:
    freq[ch] = freq.get(ch, 0) + 1

print(freq["a"])

OOP Basics (Phase 4)

Use classes for cohesive state + behavior. Keep methods focused and side effects explicit.

class Counter:
    def __init__(self):
        self.value = 0

    def add(self, step=1):
        self.value += step

c = Counter()
c.add(4)
print(c.value)

File I/O (Phase 4)

Always use context managers so resources close even if runtime errors occur.

with open("result.txt", "w", encoding="utf-8") as file:
    file.write("ZenPy practice log\n")

with open("result.txt", "r", encoding="utf-8") as file:
    print(file.read().strip())

Exceptions (All Phases)

Catch only expected failures and preserve useful messages for debugging.

raw = "42"
try:
    value = int(raw)
except ValueError:
    print("Invalid integer")
else:
    print(value)

Math & Stats (Phase 10-12)

Many new questions rely on mean, variance, probability formulas, and safe float formatting.

nums = [1, 2, 3, 4, 5]
mean = sum(nums) / len(nums)
var = sum((x - mean) ** 2 for x in nums) / len(nums)
print(f"{mean:.2f}")
print(f"{var:.4f}")

Tokenization & Vocabulary (Phase 13)

Lowercase + split is a minimal tokenizer. Build stable token-to-id maps using first occurrence order.

sentence = "Build Better Models"
tokens = sentence.lower().split()
vocab = {}
for token in tokens:
    if token not in vocab:
        vocab[token] = len(vocab)
print(tokens)
print(vocab)

Sequence Windows (Phase 14)

Create context-target pairs by shifting windows. This pattern is the backbone of next-token prediction tasks.

ids = [5, 6, 7, 8, 9]
window = 3
for i in range(len(ids) - window):
    context = ids[i:i + window]
    target = ids[i + window]
    print(context, "->", target)

Attention Basics (Phase 15)

Start from dot product, then scale by sqrt(d). Keep implementation explicit before optimization.

q = [1, 2, 3]
k = [4, 5, 6]
dot = sum(a * b for a, b in zip(q, k))
d = len(q)
scaled = dot / (d ** 0.5)
print(dot)
print(f"{scaled:.4f}")

Practice Rules

Read input carefully, print exactly expected output, use hints only when blocked, and re-run tests locally before submit. Every 10 first-time solves now grants a Zen bonus, so consistency matters.
